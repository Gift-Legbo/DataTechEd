# Puresip’s Dashboard — Reducing Workload and Aiding Automation Process Analysis

An interactive \*\*Power BI dashboard\*\* that tracks sales, retailer, regional, and brand performance for a beverage portfolio — transforming fragmented spreadsheets into a single source of truth.

!\[Dashboard Preview](https://raw.githubusercontent.com/Gift-Legbo/DataTechEd/refs/heads/main/Puresip-Dashboard/Assets/Dashboard%20Screenshot.png)

## The Problem

The company relied on individual spreadsheets managed by each regional manager to track sales. This created:

\- \*\*Data consolidation issues\*\* — no unified view of performance

\- \*\*Accuracy concerns\*\* — manual entry led to inconsistencies

\- \*\*Limited analysis\*\* — no visibility into product-level profitability or trends across locations

The goal was to use Power BI to \*\*streamline data management\*\* and surface \*\*actionable insights\*\* on sales trends and regional performance.

## The Solution

A shared data folder on a central server is connected to Power BI. Retailers upload their spreadsheets quarterly, and the dashboard refreshes automatically to reflect the latest figures — no manual intervention required.

\### Key Features

\- \*\*Executive-ready dark theme\*\* design with slicers for Beverage Brand and Contact

\- \*\*Tooltip page\*\* for on-hover product detail

\- \*\*Automatic data refresh\*\* via folder connection

\- \*\*Scalable data model\*\* built on a star schema
